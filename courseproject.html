COURSE PROJECT – HEALTH & INFRASTRUCTURE

City of Prithvi Lok – Algorithm-Driven Health System Design

This course project applies advanced Design and Analysis of Algorithms (DAA) to a fictional tier-2 smart city called Prithvi Lok. Under strict constraints of region and budget, each business case models a realistic health-infrastructure challenge and maps it to specific algorithms and data structures.

Project Constraints
Region: 150 sq km (urban + peri-urban) Budget: ₹20,000 crore (8-year horizon) Domain: Health & Civic Infrastructure Goal: SDG-aligned, algorithm-first planning
Algorithms Used (Core Set)

Only a focused set of algorithms and data structures are used across all business cases to keep the design consistent and analyzable.

BFS / DFS Dijkstra Floyd–Warshall Bellman–Ford MST (Kruskal) Fenwick Tree BST Red-Black Tree Hash Tables Trie

Business Case Studies (Prithvi Lok)

Each card represents one realistic health-infrastructure problem in Prithvi Lok, the algorithms used, and a direct link to the corresponding C++ implementation in your GitHub repository.

Case 01 · Coverage
Primary Health Center Coverage in Prithvi Lok

Service: PHC Network Planning City: Prithvi Lok

  • Problem: Ensure every ward is within 3 hops of a primary health center.
  • Algorithms: BFS (Breadth-First Search)
  • Data Structures: Adjacency list, Queue
  • Outcome: 98% population coverage with minimal new PHCs.
SDG 3 – Good Health & Well-Being View Code
Case 02 · Emergency
Ambulance Dispatch for Golden-Hour Response

Service: Emergency Transport Fleet: 48 Units

  • Problem: Reduce average ambulance response time below 8 minutes.
  • Algorithms: Dijkstra’s Algorithm
  • Data Structures: Adjacency list, Min-Heap
  • Outcome: Response time improved from 15 → 7.2 minutes.
SDG 3 – Emergency Care View Code
Case 03 · Network
Hospital-to-Hospital Referral Time Matrix

Service: Tertiary Referral Nodes: 24 Facilities

  • Problem: Identify best referral chains between any two hospitals.
  • Algorithms: Floyd–Warshall
  • Data Structures: Distance matrix (2D array)
  • Outcome: Referral delays reduced by 23% by re-routing flows.
SDG 3 – Efficient Referrals View Code
Case 04 · Outreach
High-Risk Area Outreach Planning

Service: Rural & Slum Outreach Risk-Weighted Roads

  • Problem: Reach high-risk pockets considering risk/traffic penalties.
  • Algorithms: Bellman–Ford
  • Data Structures: Edge list, distance array
  • Outcome: 31% more high-risk households visited in each campaign.
SDG 3 & SDG 10 – Reduced Inequalities View Code
Case 05 · Connectivity
Telemedicine Fiber Backbone for PHCs

Service: Digital Health Network Budget: ₹480 Cr (subset)

  • Problem: Connect 30 health nodes with minimum cabling cost.
  • Algorithms: Minimum Spanning Tree – Kruskal’s
  • Data Structures: Edge list, Disjoint Set (DSU)
  • Outcome: ~22% cost saving vs naive network layout.
SDG 3 & SDG 9 – Infrastructure View Code
Case 06 · Analytics
Real-Time Bed Occupancy & Capacity Alerts

Service: Hospital Operations 26 Facilities

  • Problem: Answer “beds free in any zone?” in sub-second time.
  • Algorithms: Fenwick Tree (Binary Indexed Tree)
  • Data Structures: Fenwick tree over bed counts
  • Outcome: Instant occupancy dashboards for command center.
SDG 3 – System Readiness View Code
Case 07 · Identity
Unified Smart Health Card Lookup

Service: Patient Identity Population: 32 Lakh

  • Problem: Fetch patient record in < 1 second at any desk.
  • Algorithms: Hashing with Open Addressing / Chaining
  • Data Structures: Hash Table
  • Outcome: Lookup time ≈ O(1), duplicates reduced significantly.
SDG 16 – Strong Institutions View Code
Case 08 · Search
Auto-Complete for Drugs & Diagnoses

Service: e-Prescription System Dictionary: 40k+ Terms

  • Problem: Fast prefix-based search while doctors type.
  • Algorithms: Trie (Prefix Tree) Traversal
  • Data Structures: Trie with frequency counters
  • Outcome: Typing time reduced, fewer prescription errors.
SDG 3 – Safe Prescriptions View Code
Case 09 · Inventory
Pharmacy Stock & Expiry Management

Service: City-Wide Pharmacies Items: 18k SKUs

  • Problem: Query “next expiring batch” and reorder thresholds quickly.
  • Algorithms: BST Operations (Insert, Delete, Search, Inorder)
  • Data Structures: Binary Search Tree
  • Outcome: Out-of-stock and wastage incidents reduced.
SDG 3 – Essential Medicines View Code
Case 10 · Scheduling
Balanced Appointment Scheduling Platform

Service: Unified Appointment System Peak Load: 10k/day

  • Problem: Maintain balanced search for free slots at high scale.
  • Algorithms: Red-Black Tree insert/delete
  • Data Structures: Red-Black Tree
  • Outcome: Stable O(log n) slot operations even at peak traffic.
SDG 3 – Access to Care View Code
