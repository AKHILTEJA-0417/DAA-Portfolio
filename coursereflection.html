Iteration, Recursion & Backtracking

Iteration

Repeating steps until a condition is met — mainly loops and traversal methods.

Recursion

Functions calling themselves — used in tree traversals, factorial, divide and conquer.

Backtracking

Exploring all possibilities by reverting steps — used in N-Queens, Sudoku, and pathfinding.

Space Efficiency

Space efficiency is the ability of an algorithm to use the least amount of memory while still producing correct results. It becomes essential when working with memory-limited environments or very large datasets.

Used In:
  • Hash Tables
  • Lookup Tables
  • Database Indexing

Space & Time Trade-off

The space–time trade-off explains how improving memory usage may increase runtime and how improving speed may require additional memory. Understanding this balance is crucial when designing highly optimized algorithms.

More Space → Faster Execution
Less Space → Slower Execution

Time Efficiency

Time efficiency measures how fast an algorithm runs relative to input size. Highly efficient algorithms reduce execution delays, especially in real-time systems and large datasets.

Used In:
  • Memoization in Dynamic Programming
  • Sorting with Extra Memory
  • Breadth-First Search (BFS)
